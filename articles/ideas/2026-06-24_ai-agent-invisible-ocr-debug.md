# AIエージェントが「見えない不具合」をどう見つけたか — PDFの透明テキストレイヤーをデバッグ可視化で解決した話

## 何を書くか

- PDFのOCR処理で埋め込まれる「透明テキストレイヤー」は、通常の操作（検索・コピー）では使われるが目には見えない
- このレイヤーがずれていても、PDFビューアでは気づきにくい（見えないから）
- AIエージェント（コード自律改修）がこの問題にどうアプローチしたか
  - **可視化**: 透明テキストを半透明赤色に変えてPNGでレンダリング → ずれを目で確認
  - **原因特定**: PDF の `/Rotate` 属性と HexaPDF の canvas 座標系のミスマッチを特定
  - **数学的解決**: CTM（Current Transformation Matrix）を導出して全回転パターンに対応
  - **検証**: サンプルPDFを作成し、全4パターン（0°/90°/180°/270°）で修正前/修正後の比較
- コードの修正量はわずか22行。しかし根本原因の特定と正しいCTMの導出が本質

## なぜ書くか

- **「見えない不具合」をAIエージェントがどう扱うか** という実践的な事例
- 人間が目視で気づきにくいバグを、AIが能動的にデバッグ手法を考案して解決する過程が面白い
- PDF内部構造（MediaBox、Rotate、CTM）の知識は汎用性が高い
- HexaPDFの座標系の罠は、同じライブラリを使う他のエンジニアにも有用

## 想定読者

- AIエージェント（Copilot, Claude, Gemini等）を開発ワークフローに導入しているエンジニア
- PDF処理（OCR、テキスト抽出、テキストレイヤー埋め込み）を扱うバックエンドエンジニア
- 「AIに任せるとどこまでやってくれるのか」に興味がある技術リーダー・CTO

## キーポイント

1. **「見えない」ものはデバッグも見えない** — 透明テキストレイヤーのバグは、通常のテスト（目視確認）では検出不能
2. **可視化がブレイクスルー** — `fill_alpha: 0.0` を `0.5` に変更し、赤色テキストとしてレンダリングするだけでバグが一目瞭然に
3. **座標変換は数学で解ける** — PDF仕様の `/Rotate` と HexaPDF の `canvas.transform` を理解すれば、CTM行列を導出できる
4. **AIエージェントの強み** — 座標変換行列の導出、サンプルPDF作成、ECSでの検証、比較画像生成を一気通貫で実行
5. **予防的網羅対応** — 発生が確認されたのは90°だけだが、180°/270°も同時に対応（追加コスト6行）

## 記事の構成案

### 1. はじめに — 「検索結果がずれている」という報告
- PDF検索で文字をハイライトすると、テキストと全く異なる位置にハイライトが出る
- でもPDFの見た目は正常。何が起きているのか？

### 2. PDFの透明テキストレイヤーとは
- OCR処理でスキャンPDFに埋め込まれる透明テキスト
- ユーザーには見えないが、検索・コピー・アクセシビリティに使われる
- このレイヤーがずれると「見た目は正常だが検索結果がおかしい」という症状になる

### 3. AIエージェントのアプローチ — 可視化でデバッグ
- 透明テキストを半透明赤色に変更し、PNGレンダリングで位置を確認
- Before/After画像の比較（ここでサンプルPDFの画像を使用）
- 90°ページ: テキストが**縦書き**になってずれている → バグ確認

### 4. 原因分析 — PDF座標系の罠
- PDF MediaBox座標系 vs 表示座標系
- `/Rotate` 属性の挙動（表示時に回転するが、内部座標は回転しない）
- HexaPDF の `canvas` は MediaBox 座標系で描画する
- Cloud Vision API は表示座標系で座標を返す → **ミスマッチ**

### 5. 修正 — CTM（座標変換行列）の導出
- 各Rotation値に対するCTM行列を数学的に導出
- `canvas.transform(a, b, c, d, e, f)` による変換
- テーブルで全4パターンを整理

### 6. 検証 — サンプルPDFで全パターンテスト
- 機密情報を含まないサンプルPDFを作成
- 修正前/修正後の比較画像（0°, 90°, 180°, 270°）
- 全パターンで位置精度を確認

### 7. まとめ — AIエージェントによるデバッグの可能性
- 「見えない不具合」への対処力
- デバッグ手法の考案能力
- 網羅的なテストの実行
